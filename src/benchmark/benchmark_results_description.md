# Анализ производительности учебной реализации свёртки

Проведён эксперимент по сравнению скорости выполнения свёртки между учебной реализацией (на языке Python с использованием вложенных циклов и NumPy) и библиотечной функцией OpenCV (cv2.filter2D), написанной на C++ с низкоуровневыми оптимизациями.

Измерения проводились для квадратных изображений трёх размеров: 128×128, 512x512 и 2048×2048 пикселей. Для каждого размера использовались несколько ядер и режимов обработки краёв.

## Результаты

- **128×128**: 
    - **Grayscale**: Educational ≈ 21.00 мс, OpenCV ≈ 0.01 мс. Разрыв в 2100 раз. 
    - **RGB**: Educational ≈ 68.00 мс, OpenCV ≈ 0.03 мс. Разрыв в 2267 раз. 
- **1024×1024**:
    - **Grayscale**: Educational ≈ 0.34 с, OpenCV ≈ 0.15 мс. Разрыв в 2267 раз. 
    - **RGB**: Educational ≈ 1.15 с, OpenCV ≈ 0.44 мс. Разрыв в 2614 раз. 
- **4096×4096**:
    - **Grayscale**: Educational ≈ 5.75 с, OpenCV ≈ 2.79 мс. Разрыв в 2061 раз. 
    - **RGB**: Educational ≈ 17.52 с, OpenCV ≈ 12,72 мс. Разрыв в 1377 ыраз. 

## Причины различий

Учебная реализация выполняется в интерпретируемой среде Python. Каждая операция внутри циклов сопровождается накладными расходами на вызовы интерпретатора и управление памятью.

Библиотека OpenCV скомпилирована в машинный код, использует SIMD-расширения процессоров (AVX, SSE) и автоматически распараллеливает вычисления на все доступные ядра. Кроме того, внутренние алгоритмы оптимизированы по доступу к кэш-памяти.

## Вывод

Эксперимент демонстрирует, что учебная реализация свёртки является непригодной для обработки изображений большого размера в практических приложениях из‑за низкой производительности. Однако она полностью выполняет свою главную задачу - служит наглядной демонстрацией алгоритма свёртки и помогает понять принципы работы.
